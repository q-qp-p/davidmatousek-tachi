# templates/security-report/

Typst rendering templates for the `/security-report` command. These templates are compiled by the Typst CLI to produce a professional multi-page PDF security assessment booklet.

## File Inventory

| File | Purpose |
|------|---------|
| `main.typ` | Master orchestrator — imports all templates and report data, conditionally includes pages |
| `shared.typ` | Shared design tokens: severity colors, typography, page geometry, header/footer functions |
| `cover.typ` | Cover page: project name, date, classification, risk posture, severity counts |
| `executive-summary.typ` | Executive summary: 2-column rich mode (with narrative) or single-column minimal mode |
| `full-bleed.typ` | Full-bleed infographic page: 16:9 landscape, zero margins, reused for all 3 image types |
| `findings-detail.typ` | Findings table: 3-tier column sets, severity-sorted, multi-page overflow with repeated headers |
| `control-coverage.typ` | Control coverage: STRIDE status matrix, detailed control table, summary statistics |
| `remediation-roadmap.typ` | Remediation roadmap: prioritized actions grouped by severity with SLA targets |

## How It Works

1. The **report-assembler agent** (`.claude/agents/tachi/report-assembler.md`) parses tachi pipeline artifacts
2. It generates `report-data.typ` containing all extracted data as Typst `#let` variable bindings
3. `main.typ` imports `report-data.typ` and `shared.typ`, then conditionally includes each page template
4. The Typst CLI compiles `main.typ` into a single PDF: `typst compile main.typ output.pdf --root .`
5. The intermediate `report-data.typ` is cleaned up after successful compilation

## Data Contract

`report-data.typ` must export these variables (generated by the agent):

- **Metadata**: `project-name`, `assessment-date`, `classification`
- **Severity counts**: `critical-count`, `high-count`, `medium-count`, `low-count`, `total-findings`
- **Page flags**: `has-threat-report`, `has-risk-scores`, `has-compensating-controls`, `has-funnel-image`, `has-baseball-image`, `has-architecture-image`
- **Data tier**: `data-source-tier` (1 = residual risk, 2 = inherent risk, 3 = severity)
- **Image paths**: `funnel-image-path`, `baseball-image-path`, `architecture-image-path` (relative to this directory)
- **Content**: `executive-narrative`, `component-distribution`, `findings`, `coverage-matrix`, `controls`, `coverage-summary`, `remediation-actions`

## Severity Color Palette

| Severity | Color | Hex |
|----------|-------|-----|
| Critical | Red | `#DC2626` |
| High | Orange | `#F97316` |
| Medium | Yellow | `#EAB308` |
| Low | Blue | `#4169E1` |

## Typography

- **Headings**: Sans-serif (Helvetica Neue > Helvetica > Arial > New Computer Modern)
- **Body**: Serif (New Computer Modern > Charter > Georgia > Times New Roman)
- **Tables**: Monospace (Menlo > Courier New > Courier) for data cells

No custom font files required — uses Typst built-in and system font stacks.

## Typst Version

Tested with Typst 0.14.2. Should work with 0.11.x+.

Install: `brew install typst` (macOS) | `cargo install typst-cli` (Linux) | `winget install typst` (Windows)

## Manual Compilation

To compile manually (for testing or debugging):

```bash
# From the project root:
typst compile templates/security-report/main.typ output.pdf --root .

# Note: report-data.typ must exist in this directory before compilation.
# It is normally generated by the report-assembler agent.
```

## Schema Reference

Page assembly rules are defined in `schemas/security-report.yaml`. The schema declares page types, sequence order, artifact-to-page mappings, data source tiers, and layout specifications.
