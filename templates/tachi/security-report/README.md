# templates/tachi/security-report/

Typst template system for the `/tachi.security-report` command. These templates are compiled by the Typst CLI to produce a professional multi-page PDF security assessment booklet with tachi branding.

## Page Types

The report renders up to 12 pages in fixed sequence. Conditional pages are gated by data availability or configuration flags.

| # | Page | File | Condition |
|---|------|------|-----------|
| 1 | Cover | `cover.typ` | Always |
| 2 | Disclaimer | `disclaimer.typ` | `show-disclaimer` (default: true) |
| 3 | Table of Contents | `toc.typ` | Always (auto-generated from headings) |
| 4 | Risk Methodology | `methodology.typ` | `show-methodology` (default: true); adapts sections based on `has-risk-scores` and `has-compensating-controls` |
| 5 | Assessment Scope | `scope.typ` | Always; renders from threats.md Sections 1-2 |
| 6 | Executive Summary | `executive-summary.typ` | Always; rich mode (with narrative) or minimal mode (severity counts only) |
| 7 | Risk Reduction Funnel | `full-bleed.typ` | `has-funnel-image` |
| 8 | Baseball Card | `full-bleed.typ` | `has-baseball-image` |
| 9 | System Architecture | `full-bleed.typ` | `has-architecture-image` |
| 10 | Findings Detail | `findings-detail.typ` | Always; card-based layout with 3 data tiers |
| 11 | Control Coverage | `control-coverage.typ` | `has-compensating-controls` |
| 12 | Remediation Roadmap | `remediation-roadmap.typ` | `has-compensating-controls` or threat report with remediation data |

## Theme Customization

Edit `theme.typ` to rebrand the report. All page templates consume theme tokens through `shared.typ` -- no other file contains hardcoded structural colors.

**Brand color tokens** (7):

| Token | Default | Usage |
|-------|---------|-------|
| `brand-primary` | `#1B2A4A` (Tachi Indigo) | Headers, primary surfaces, cover title |
| `brand-secondary` | `#2D4A6F` (Steel Blue) | Secondary/supporting elements |
| `brand-accent` | `#C93A40` (Vermillion) | Classification banners, accent lines |
| `brand-highlight` | `#B8963E` (Muted Gold) | Premium accent touches |
| `brand-dark` | `#0D0D0D` (Lacquer Black) | Body text on dark backgrounds |
| `brand-light` | `#F5F2EB` (Washi White) | Page backgrounds, card fills |
| `brand-muted` | `#64748B` (Ash Gray) | Secondary text, captions, footers |

Changing `brand-primary` propagates to all structural elements: header backgrounds, title colors, heading fills, and cover surfaces.

**Logo path tokens** (2):

| Token | Usage |
|-------|-------|
| `logo-primary-path` | Vertical lockup on cover page |
| `logo-horizontal-path` | Horizontal lockup in page headers |

**Font stack tokens** (3):

| Token | Default | Usage |
|-------|---------|-------|
| `font-heading` | Helvetica Neue > Helvetica > Arial > New Computer Modern | Section headings |
| `font-body` | New Computer Modern > Charter > Georgia > Times New Roman | Body text |
| `font-mono` | Menlo > Courier New > Courier | Data cells, code |

No custom font files required -- uses Typst built-in and system font stacks.

## Configuration

Edit `report-config.typ` to control page visibility and custom text without modifying templates.

| Variable | Type | Default | Effect |
|----------|------|---------|--------|
| `show-disclaimer` | boolean | `true` | Include the disclaimer page (page 2) |
| `show-methodology` | boolean | `true` | Include the risk methodology page (page 4) |
| `custom-disclaimer-text` | string or none | `none` | Replaces all four default notice sections |
| `custom-footer-text` | string or none | `none` | Replaces "Generated by tachi" in page footers |

The report assembler copies a user-provided `report-config.typ` from the target directory before compilation. If no user config exists, the default file is used.

## Architecture

### Import Chain

```
theme.typ  -->  shared.typ  -->  page templates (cover.typ, scope.typ, ...)
                                      ^
report-data.typ ---+                  |
                   +--- main.typ -----+
report-config.typ -+
```

- `theme.typ` defines brand tokens (colors, logos, fonts)
- `shared.typ` imports theme tokens and exports structural aliases, severity colors, typography rules, header/footer functions, and page geometry
- `main.typ` imports `shared.typ`, `report-data.typ`, and `report-config.typ`, then calls each page template function in sequence
- Page templates import `shared.typ` and receive data as function parameters from `main.typ`

### How It Works

1. The **report-assembler agent** (`.claude/agents/tachi/report-assembler.md`) parses tachi pipeline artifacts
2. It generates `report-data.typ` containing all extracted data as Typst `#let` variable bindings
3. `main.typ` imports `report-data.typ`, `report-config.typ`, and `shared.typ`, then conditionally includes each page template
4. The Typst CLI compiles `main.typ` into a single PDF: `typst compile main.typ output.pdf --root .`
5. The intermediate `report-data.typ` is cleaned up after successful compilation

### Data Contract

`report-data.typ` must export these variables (generated by the agent):

- **Metadata**: `project-name`, `assessment-date`, `classification`
- **Severity counts**: `critical-count`, `high-count`, `medium-count`, `low-count`, `total-findings`
- **Page flags**: `has-threat-report`, `has-risk-scores`, `has-compensating-controls`, `has-funnel-image`, `has-baseball-image`, `has-architecture-image`
- **Data tier**: `data-source-tier` (1 = residual risk, 2 = inherent risk, 3 = severity)
- **Image paths**: `funnel-image-path`, `baseball-image-path`, `architecture-image-path`
- **Scope data**: `scope-components`, `scope-data-flows`, `scope-trust-boundaries`, `scope-boundary-crossings`, `scope-component-count`, `scope-data-flow-count`, `scope-trust-boundary-count`
- **Brand assets**: `has-logo-primary`, `has-logo-horizontal`, `logo-primary-path`, `logo-horizontal-path`
- **Page visibility**: `show-disclaimer`, `show-methodology`
- **Content**: `executive-narrative`, `component-distribution`, `findings`, `coverage-matrix`, `controls`, `coverage-summary`, `remediation-actions`

## Breaking Changes from v1.0 (PRD-054)

- **`report-data.typ` requires additional variables**: scope data (7 variables), brand asset flags and paths (4 variables), and page visibility flags (2 variables). See the data contract above for the complete list.
- **`report-data.typ` is auto-generated and transient**: It is produced by the report assembler and deleted after compilation. Backward compatibility applies to the pipeline interface (report-assembler agent), not the intermediate file.
- **New import: `report-config.typ`**: Required alongside `report-data.typ`. Provides user-configurable overrides for page visibility and custom text. A default file is included in this directory.

## Severity Color Palette

| Severity | Color | Hex |
|----------|-------|-----|
| Critical | Red | `#DC2626` |
| High | Orange | `#F97316` |
| Medium | Yellow | `#EAB308` |
| Low | Blue | `#4169E1` |

Severity colors are functional constants in `shared.typ`, not brand tokens. They remain fixed regardless of theme customization.

## Typst Version

Tested with Typst 0.14.2. Should work with 0.11.x+.

Install: `brew install typst` (macOS) | `cargo install typst-cli` (Linux) | `winget install typst` (Windows)

## Manual Compilation

```bash
# From the project root:
typst compile templates/tachi/security-report/main.typ output.pdf --root .

# Note: report-data.typ must exist in this directory before compilation.
# It is normally generated by the report-assembler agent.
```

## Schema Reference

Page assembly rules are defined in `schemas/security-report.yaml`. The schema declares page types, sequence order, artifact-to-page mappings, data source tiers, and layout specifications.
