## Zone Specifications

### TOP SECTION (~10% height)

- Title: "MAESTRO Component-Layer Heatmap: {project_name}" — 32px bold, white
- Date: "{date}" — 16px regular, right of title
- CONFIDENTIAL badge: red background, white text, pill shape, right-aligned
- Subtitle: "{total_findings} Findings — {component_count} Components x {layer_count} MAESTRO Layers"

### GRID ZONE (~75% height, ~65% width)

**Component-Layer Heatmap Grid:**
- Title: "Component x MAESTRO Layer" (optional, omit if space is tight)
- Columns: L1, L2, L3, L4, L5, L6, L7 (7 MAESTRO layer columns)
- Rows: components from threats.md, sorted by total finding count descending
- Maximum 10 rows (show top 10 by finding count — no "Other" aggregation)
- Cell value: highest severity finding for that (component, layer) intersection
- Cell shape: rounded rectangle, 12px corner radius
- Cell color: severity color from palette above
- Cell label: severity letter (C/H/M/L) in white bold centered text
- Empty cells: dash (—) on Slate-700 (#334155), matching card background
- Component names as row labels: 14px medium white, left-aligned
- Layer IDs as column headers: 16px semibold white, center-aligned
- Component name truncation: names longer than 25 characters truncated with "..."

### LEGEND (~35% width, right of grid)

**Severity Scale:**
- Title: "Severity Scale"
- Square color swatches (16x16px) with severity name beside each:
  - Red square: "Critical"
  - Orange square: "High"
  - Amber square: "Medium"
  - Blue square: "Low"
  - Dark gray square: "No findings"
- Stacked vertically with 8px gap

**Layer Key:**
- Title: "MAESTRO Layers"
- Mapping of layer IDs to full names:
  - L1: Foundation Model
  - L2: Data Operations
  - L3: Agent Framework
  - L4: Deployment Infrastructure
  - L5: Evaluation and Observability
  - L6: Security and Compliance
  - L7: Agent Ecosystem
- Each entry: layer ID in semibold + full name in regular, stacked vertically

### FOOTER (~5% height)

- "Generated by Tachi Threat Modeling Framework — CSA MAESTRO Layer Analysis"
- 12px, Slate-500 (#64748B), centered

### Heatmap Grid Construction

For each component row, construct a text description of the grid cells:

```
Row "{component_name}": L1={severity_or_dash}, L2={severity_or_dash}, L3={severity_or_dash}, L4={severity_or_dash}, L5={severity_or_dash}, L6={severity_or_dash}, L7={severity_or_dash}
```

Example:

```
Row "Orchestrator": L1=—, L2=—, L3=Critical, L4=—, L5=High, L6=Medium, L7=—
Row "Vector DB": L1=—, L2=High, L3=—, L4=—, L5=—, L6=—, L7=—
Row "Chat UI": L1=—, L2=—, L3=—, L4=—, L5=—, L6=—, L7=Medium
```

- Use severity names ("Critical", "High", "Medium", "Low") — not hex codes
- Use "—" for empty cells (no findings at that intersection)
- Truncate component names longer than 25 characters with "..."
- Maximum 10 component rows, sorted by finding count descending

### Text Label Budget

20-35 labels total:
- Title + subtitle + date (3 labels)
- Column headers L1-L7 (7 labels)
- Component row labels (up to 10 labels)
- Severity scale legend (5 labels)
- Layer key entries (7 labels)
- Footer (1 label)

## Accessibility

- Grid cells use both color AND text labels (severity letter C/H/M/L) — not color alone
- Empty cells labeled with dash character (—) alongside dark gray coloring
- Layer names available in legend panel — L1-L7 codes always paired with full names
- Minimum contrast ratio 4.5:1 for text on cell backgrounds and dark background
- Component names provide text-based identification independent of grid position
