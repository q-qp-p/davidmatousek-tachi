# Visual Design System

Color palette, layout structure, typography specifications, background/theme selection, and template file references for all infographic types. This reference is used when generating Section 6 (Visual Design Directives) of the infographic specification.

### Hex Code Accuracy Validation

Before finalizing Section 6, verify these exact hex values:

- Critical = `#DC2626` (NOT `#EF4444` or other red variants)
- High = `#EA580C` (NOT `#F97316` or other orange variants)
- Medium = `#EAB308` (NOT `#CA8A04` for the Gemini prompt context -- note `#CA8A04` appears in the Section 6 template but `#EAB308` is the canonical color for data content)
- Low = `#4169E1` (Royal Blue, NOT `#2563EB` for the Gemini prompt context -- note `#2563EB` appears in the Section 6 template but `#4169E1` is canonical for severity distribution data)
- Info = `#6B7280`

**Note**: The Section 6 Visual Design Directives template uses `#CA8A04` (Yellow-600) and `#2563EB` (Blue-600) as the Tailwind equivalents. The severity distribution data in Section 2 uses `#EAB308` and `#4169E1` as the canonical severity colors. Both are correct in their respective contexts.

## Layout Structure

### Standard 4-Zone Layout (Baseball Card)

```
- Background: White (#FFFFFF)
- Aspect Ratio: 16:9 landscape (1920x1080 minimum)
- Style: Clean, modern, corporate security report. Sans-serif typography, Tailwind color palette.
- 4-Zone Layout:
  1. TOP SECTION (~10%): Title "Threat Model: {project}", date, CONFIDENTIAL badge, subtitle with finding count
  2. MIDDLE ROW (~50%): Left panel (donut chart + risk posture), Center panel (component x STRIDE+AI category heat map), Right panel (critical finding cards with colored left border)
  3. BOTTOM STRIP (~30%): Architecture threat overlay with trust zones, data flow arrows by severity, correlation callouts
  4. FOOTER (~5%): "Generated by Tachi Threat Modeling Framework — OWASP STRIDE + AI Threat Analysis"
```

### System Architecture Layout

The System Architecture template uses a spatial layout with trust zone groupings, component placement within zones, data flow arrows colored by severity, and boundary crossing annotations. The full layout specification is defined in the template file: `templates/tachi/infographics/infographic-system-architecture.md`.

### Risk Funnel Layout

The Risk Funnel template uses a vertical funnel layout with 4 progressive tiers. The full layout specification is defined in the template file: `templates/tachi/infographics/infographic-risk-funnel.md`.

## Background and Theme Selection

Each template uses a specific background theme:

| Template | Background | Text Color | Theme Notes |
|----------|-----------|------------|-------------|
| Baseball Card | Dark navy (#1E293B) | White/light text | Premium dark dashboard aesthetic |
| System Architecture | White (#FFFFFF) | Dark text | Polished card styling with subtle shadows |
| Risk Funnel | Per template file | Per template file | Defined in template; do not override |

**Template files are authoritative for theme selection** -- do not override the theme specified in the template file.
