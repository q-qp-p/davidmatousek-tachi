# =============================================================================
# AOD-kit Template Manifest
# =============================================================================
#
# Feature 129 (downstream template update mechanism) — PLSK source of truth.
#
# Purpose:
#   Categorize every git-tracked file in this repo by ownership. Drives
#   /aod.update decisions (overwrite, preserve, re-substitute, or ignore)
#   and the CI coverage check at .github/workflows/manifest-coverage.yml.
#
# Line format:
#   <category>|<path-or-glob>
#
#   Blank lines and lines starting with `#` (optionally preceded by
#   whitespace) are skipped. Separator is a literal `|` (pipe). No spaces
#   around the separator. No trailing whitespace.
#
# Valid categories (6):
#   owned         Template content. Overwritten verbatim from upstream on
#                 /aod.update. Adopters MUST NOT hand-edit these files.
#   personalized  Template content with {{PLACEHOLDER}} tokens. Overwritten
#                 from upstream and then re-substituted from
#                 .aod/personalization.env. Adopters MUST NOT hand-edit.
#   user          Adopter-owned. Never touched by /aod.update (hidden from
#                 preview entirely). Counts toward coverage.
#   scaffold      Template provides a starter; adopter may add to it. P0:
#                 listed for coverage only; runtime copy-if-missing deferred
#                 to PRD 129b.
#   merge         Both sides edit; /aod.update previews a diff and warns
#                 (P0: warn-and-skip; 3-way merge deferred to PRD 129b).
#   ignore        Never updated and excluded from the coverage check's
#                 "must be covered" rule. Used for transient runtime state,
#                 build artifacts, and third-party directories we DO track
#                 in git but never sync.
#
# Precedence (highest wins, applied per matching file):
#   ignore > guard > user > scaffold > merge > personalized > owned
#
#   The `guard` rank is reserved for the hardcoded user-owned guard list
#   (FR-007); it does not appear in this file. Manifest entries never write
#   through guards.
#
# Anchored whole-path glob semantics:
#   `**` matches any sequence of characters, including `/`.
#   `*`  matches any sequence WITHIN a single path segment (no `/`).
#   Patterns are anchored: `CLAUDE.md` matches only the repo-root file, NOT
#   `docs/CLAUDE.md`. Use `**/CLAUDE.md` for the recursive form.
#
# No-code-from-upstream policy (FR-012):
#   Only `owned` and `personalized` categories receive content from
#   upstream. `user`, `scaffold`, `merge`, and `ignore` are never written
#   by /aod.update in P0 (scaffold/merge runtime behavior is deferred to
#   PRD 129b).
#
# References:
#   - specs/129-downstream-template-update/contracts/manifest-schema.md
#   - specs/129-downstream-template-update/plan.md §C4 (guard list), §C6 (CI)
#   - docs/guides/DOWNSTREAM_UPDATE.md (adopter walkthrough; P1)
#
# Maintainer workflow:
#   1. Run `scripts/check-manifest-coverage.sh` locally before committing
#      any new tracked file. Exit 0 = coverage complete.
#   2. The CI workflow (manifest-coverage.yml) runs the same check on every
#      push + PR to main using a bash:3.2 Docker container to enforce the
#      bash 3.2 compatibility constraint on the validator itself.
#   3. Optional pre-commit hook documented in CONTRIBUTING.md.
#
# =============================================================================
