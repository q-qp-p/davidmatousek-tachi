## Platform Adapters (Feature 021)

Feature 021 introduced platform adapters that package tachi's 14 threat agents for use across AI coding platforms and CI pipelines. These live in the `adapters/` directory at the repository root. Features 035, 036, and 039 extended the Claude Code adapter with command files for tachi's four product commands: `/threat-model`, `/risk-score`, `/compensating-controls`, and `/infographic`.

### Adapter Directory Structure

```
adapters/
  claude-code/     # .md agents with name/description frontmatter
    agents/        #   14 threat agents + risk-scorer
    commands/      #   /threat-model, /risk-score, /infographic
  copilot/         # .agent.md + .instructions.md files
  cursor/          # .mdc rule files
  generic/         # Numbered .md prompt files (any LLM)
  github-actions/  # GitHub Actions workflow with SARIF output
```

Each adapter includes a `README.md` (installation instructions) and a `VERSION` file (source commit SHA and file checksums).

### Infrastructure Impact

Platform adapters are **static files only** -- no infrastructure changes, no new services, no new environment variables required. They are copied into target projects by the user.

The one exception is the GitHub Actions adapter, which requires:
- An `LLM_API_KEY` repository secret (Anthropic API key) in the consuming repository
- GitHub Advanced Security enabled for SARIF upload to Code Scanning

See [CI/CD Guide](CI_CD_GUIDE.md) for GitHub Actions adapter details.

### VERSION Tooling

Adapter VERSION files are generated by `scripts/generate-adapter-version.sh`. This script:
- Computes the Git commit SHA at HEAD
- Generates SHA-256 checksums of source agent files
- Writes a YAML manifest to `<adapter-dir>/VERSION`

```bash
# Generate VERSION for a single adapter
./scripts/generate-adapter-version.sh adapters/claude-code

# Generate VERSION for all adapters
for dir in adapters/claude-code adapters/copilot adapters/cursor adapters/generic adapters/github-actions; do
  ./scripts/generate-adapter-version.sh "$dir"
done
```

VERSION files should be regenerated after any change to source agents in `agents/`.

## Deployment Policy (MANDATORY)

**ALL deployments MUST go through the devops agent.**

Before deploying to ANY environment:
1. Invoke devops agent (never run deploy commands directly)
2. DevOps reads: `docs/architecture/04_deployment_environments/{env}.md`
3. DevOps reads: `docs/devops/{01_Local|02_Staging|03_Production}/README.md`
4. DevOps outputs verification summary
5. Only then proceed with deployment

**Never deploy without verification** - Mismatched targets can cause data loss or service disruption.
