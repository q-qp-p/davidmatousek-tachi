Note: the 27-case contract test surface is rendered as 29 parametrized
entries because case 26 is split into 26a/26b for clarity (separate
positive + negative assertions on the same lower-mode behavior).

One implementation note worth surfacing during architect re-review at
T054: the original Step 5 contract said "the regex implicitly excludes
embedded newlines (the read already split lines), and embedded NUL
(bash regex stops at NUL)". The latter is incorrect ŌĆö bash command
substitution silently truncates content at the first NUL byte, so the
regex never sees the NUL at all. Without explicit NUL detection, the
adversarial fixture `KEY=foo<NUL>bar\n` would be loaded as `KEY=foobar\n`
(one valid KV line), bypassing the rejection promise in FR-005 AC-5.4.

The library implements an explicit Step 2b NUL pre-check that compares
`wc -c` against `tr -d '\000' | wc -c` ŌĆö any difference signals NUL
presence and rejects with exit 8 + stderr "NUL byte". This idiom is
bash 3.2 + BSD-coreutils compatible; it does not rely on `grep -q $'\x00'`
which is unreliable because bash collapses embedded NULs in argv to
empty strings (matching everything).

This Step 2b is a small but material elaboration of the FR-005 AC-5.4
"the regex implicitly rejects newline / NUL" wording. It does not
break the contract (the contract guarantees rejection; the Step 2b is
the mechanism that delivers the rejection). T054 should fold this
detail into ADR-040 ┬¦Decision Item 1 + ┬¦Consequences as a clarification
to the contract.

Absolute cost framing (for PM/Architect re-scope discussion):
- T005 baseline: 9-10ms p50 per invocation (`bash -c 'source <fixture>'`)
- T013 post-impl: 28-37ms p50 per invocation (`bash -c 'source <library>;
  aod_template_load_kv_file ...'`)
- Absolute increase: ~20-27ms per call
- For init.sh (one invocation per adopter setup): 20-27ms is invisible
  to humans (adopter perception threshold ~100ms).
- For /aod.update (one invocation per weekly run): same.
- For automated test loops (e.g., 100 unit-test invocations of the
  function): additive ~20ms ├Ś 100 = 2 seconds ŌĆö perceptible in CI but
  small.

Cost decomposition (mini-investigation):
- `bash -c 'source <library-only>'`: ~13ms p50 (library file is 200
  lines of bash with regex compilation and function definition)
- `bash -c 'source <fixture-only>'`: ~10ms p50 (T005 baseline shape)
- Function execution itself (minus library source): ~17-25ms (regex
  pass + whitelist scan + printf -v assignments)

The dominant cost driver is the library-source overhead (which does NOT
exist in the T005 baseline because T005's "library" is implicitly the
fixture file itself). In production, the library is sourced ONCE per
init.sh run ŌĆö the per-call cost in real adopter usage is much closer to
T005's baseline + the function-only ~15ms increment.

Recommendation for PM/Architect re-scope decision:
1. The benchmark methodology (fresh `bash -c` per call) AMORTIZES
   library-source cost over 1 call, which inflates the relative delta.
2. In real usage, the library is sourced once and the function is called
   multiple times ŌĆö actual per-call cost converges toward T005 baseline
   + ~15ms (still in the 5-25% tier on p95 absolute terms, well above
   25% on p50).
3. Absolute per-call cost remains <50ms in worst case ŌĆö still well
   below adopter perception threshold.
4. Q-5 Option a (accept-and-document, loosen NFR-004 to 50% with
   security tradeoff rationale) is the natural disposition; the
   methodology asymmetry is the substantive observation.

Proposed disposition (pending Architect/Team-Lead/PM concurrence):
- ADR-040 ┬¦Consequences documents the methodology asymmetry +
  amortized-vs-per-call distinction explicitly.
- NFR-004 threshold remains formally at 25% but ADR-040 records the
  one-time-per-process cost as the dominant driver.
- T054 finalizes ADR-040 ┬¦Consequences with these numbers.

    interrupt; cleared on every return path.
  - `wait $clone_pid; clone_rc=$?`.
  - Cleanup post-wait: `kill -9 $watchdog_pid 2>/dev/null; wait $watchdog_pid
    2>/dev/null` (SIGKILL because bash 3.2 SIGTERM-of-subshell may not
