# Research Summary: MAESTRO Infographic Templates and PDF Report Section

## Knowledge Base Findings
- No existing KB entries for MAESTRO visualization, infographic templates, or PDF report sections
- This is a novel feature with no prior patterns to reference

## Codebase Analysis

### Existing Infographic Template Pattern
- **Location**: `templates/tachi/infographics/infographic-{name}.md`
- **Three existing templates**: baseball-card, system-architecture, risk-funnel
- **Output naming**: `threat-{name}-spec.md` (spec), `threat-{name}.jpg` (image)
- **Alias**: `corporate-white` maps to `baseball-card`
- **Default**: No template specified generates both baseball-card and system-architecture

### Mandatory Template Sections
Every infographic template must contain:
1. Layout (ASCII zone diagram + proportions)
2. Style table (background, aspect ratio, aesthetic)
3. Color Palette (hex codes for 5 severity levels)
4. Typography table
5. Zone Specifications (per-zone content and data bindings)
6. Gemini Prompt Template (`{placeholder}`-based, STYLING DIRECTIVES / DATA CONTENT)
7. Gemini API Configuration (model, fallback, response modalities, aspect ratio, image size)
8. Accessibility notes

### Spec Section Structure (shared Sections 1-4)
- Section 1: Metadata (project, date, agents, findings, risk posture)
- Section 2: Risk Distribution (severity counts, LRM-corrected percentages)
- Section 3: Coverage Heat Map (component x severity grid)
- Section 4: Top Critical Findings (up to 5)
- Section 5: Template-specific (varies per template)
- Section 6: Visual Design Directives

### Extraction Script (`scripts/extract-infographic-data.py`)
- **Tier strategy**: Tier 1 (compensating-controls.md) > Tier 2 (risk-scores.md) > Tier 3 (threats.md)
- **Current MAESTRO support**: ZERO — no MAESTRO extraction exists
- `parse_scope_data` reads Components table but ignores `MAESTRO Layer` column
- `parse_threats_findings` reads Section 7 (no MAESTRO column)
- CLI `--template` choices: `baseball-card`, `system-architecture`, `risk-funnel`
- JSON output: `metadata`, `severity_distribution`, `heat_map`, `top_findings`, `template_data`

### Typst Template System (`templates/tachi/security-report/`)
- **Data injection**: `report-data.typ` (runtime-generated) imported by `main.typ`
- **Page pattern**: Each `.typ` exports a single function; `main.typ` provides `#page()` block
- **Conditional inclusion**: `#if has-{thing}` boolean gates
- **Backward compat defaults**: `#let x = if x != none { x } else { false }` pattern (lines 67-80 of main.typ)
- **Current page sequence**: Cover → Disclaimer → TOC → Methodology → Scope → Executive Summary → Risk Funnel → Baseball Card → System Architecture → Section Divider: "Detailed Findings" → Findings Detail → Control Coverage → Remediation Roadmap
- **Infographic page function**: `full-bleed.typ` `infographic-page(image-path, section-name, classification, description)`

### MAESTRO Data in threats.md
- **Section 1**: Components table has `MAESTRO Layer` column (e.g., `L1 — Foundation Model`)
- **Section 3**: Per-agent finding tables include `MAESTRO Layer` column
- **Section 6**: "Risk by MAESTRO Layer" subsection with `MAESTRO Layer | Finding Count | Highest Severity` table
- **Section 7**: Does NOT include MAESTRO Layer column

### Infographic Skill Dispatch
- **Location**: `.claude/skills/tachi-infographics/`
- **Template dispatch**: Loads `infographic-{name}.md`, produces `threat-{name}-spec.md` + `threat-{name}.jpg`
- **Reference files**: infographic-specifications.md (Sections 1-4), template-specific-formats.md (Section 5), gemini-prompt-construction.md, visual-design-system.md

### Report Assembly Skill
- **Location**: `.claude/skills/tachi-report-assembly/`
- **Contract**: `references/typst-template-contract.md`
- **New pages require**: boolean flag, data variables, conditional import in `main.typ`
- **Image paths**: `../../{target_dir}/threat-{name}.jpg` relative to security-report dir

## Architecture Constraints
- **Finding schema** (`schemas/finding.yaml`): `maestro_layer` is v1.2 field, enum of L1-L7 + "Unclassified", default "Unclassified"
- **ADR-014**: Governs optional Gemini image generation — new templates must follow optional/fallback pattern
- **Template architecture**: All new templates must follow identical structure to existing ones
- **Typst contract**: All finding dict values must be strings (quoted), even numerics
- **report-data.typ**: Generated by `extract-report-data.py`, not hand-edited

## Industry Research
- CSA MAESTRO Framework (Feb 2025): Seven-layer taxonomy for agentic AI security
- Layer visualization is common in security architecture (e.g., OSI model diagrams, NIST CSF tiers)
- Heatmap visualizations are standard for component-risk intersections in security dashboards
- Stack diagrams with severity coloring follow established infosec reporting patterns

## Recommendations for Spec
- Follow existing `infographic-{name}.md` template structure exactly for both new templates
- MAESTRO extraction should parse Section 6 for aggregate data and Section 1/3 for per-finding mapping
- Typst page should follow `findings-detail.typ` single-export-function pattern
- Heatmap should cap components at top N (consistent with existing heat_map cap of 7+Other)
- `maestro` shorthand should be handled at skill dispatch level, not CLI parser
- Backward compatibility via `has-maestro-data` boolean gate is consistent with existing pattern
- New report-data.typ variables need default values in main.typ lines 67-80 pattern area
